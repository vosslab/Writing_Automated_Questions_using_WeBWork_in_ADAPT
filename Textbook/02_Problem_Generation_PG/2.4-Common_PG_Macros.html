This page is not a catalog of everything PG can do. It is a short list of macros you will actually see and use when writing PGML-first science problems.

Load macros only once per file, list only what you need, and let dependencies load themselves. When in doubt, start small and add a macro only when the interaction or grading behavior demands it.

Always load (core macros)

Macro What it enables What breaks if it is missing Notes
PGstandard.pl Core PG helpers and defaults. Basic PG functions are undefined. Load first.
MathObjects.pl MathObjects and robust answer checking. Answer objects and checkers fail to build. Often loaded by other macros, but safe to include.
PGML.pl PGML blocks, formatting, and answer blanks. PGML blocks fail or render as raw text. Required for PGML-first prompts.
PGcourse.pl Course-level helpers and defaults. Course-specific helpers are undefined. Conventionally load last.

Interaction-specific macros (load only when needed)

What students see Macro to load What breaks if it is missing
Single-choice questions (radio buttons) parserRadioButtons.pl Radio button widgets fail to render.
Multiple selection questions (checkbox lists) parserCheckboxList.pl Checkbox list widgets fail to render.
Matching or shuffled choice lists PGchoicemacros.pl Matching and choice helpers fail to build.

Science-native contexts and grading helpers

Use it when Macro to load Why it matters
Units are part of correctness contextUnits.pl Supports unit-aware parsing and validation.
You need chemical reaction input contextReaction.pl Validates reactants/products and reaction structure.
Multiple blanks depend on each other parserMultiAnswer.pl Ties related blanks to one checker.
Parts should contribute different weights weightedGrader.pl Supports weighted scoring across parts.
Order should not matter in a list unorderedAnswer.pl Grades set-style answers without order.
You want targeted feedback for common errors answerHints.pl Adds instructor-style feedback without extra widgets.

Tables, layouts, and visual tools

Use it when Macro to load Why it matters
You need structured tables or aligned blanks niceTables.pl / unionTables.pl Keeps data displays clean and readable.
You want scaffolded, multi-part prompts scaffold.pl Supports structured sections without custom logic.
An interactive graph tool is required parserGraphTool.pl Provides interactive graph widgets.

Recognition-only legacy macros

You will see these in older problems. You do not need them for PGML-first work.

  • Algorithmic question helpers: patterns for controlled randomization in older problem banks.
  • Inexact values (significant figures): the InexactValue macro and related helpers for sig-fig-aware grading. Use only when your course truly requires strict sig-fig handling.

Safe removal checklist

  • Does the problem still render without errors?
  • Do answer blanks still appear?
  • Does the interaction type still work as expected?

Minimal PGML-first loadMacros block

DOCUMENT();

loadMacros(
	"PGstandard.pl",
	"MathObjects.pl",
	"PGML.pl",
	"PGcourse.pl",
);

Rule of thumb

  • Start with the minimal macro set above.
  • Add one extra macro only when you need a specific interaction type or context.
  • Keep the setup short; most authoring should happen in PGML.
