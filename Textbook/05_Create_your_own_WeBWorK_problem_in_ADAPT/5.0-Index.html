This chapter focuses on the ADAPT workflow: creating, finding, cloning, and assigning WeBWorK questions. The source archive includes several short video-based guides; this chapter turns that into a written checklist you can follow while building a question bank.

Workflow overview

  1. Create or clone a question into your personal question area.
  2. Edit the code and metadata.
  3. Test and revise until the prompt, answers, and scoring behave correctly.
  4. Add the question to an assignment.
  5. Reorder questions, adjust points, and publish.

Practical checklist for a new WeBWorK question

  • Start from a working PGML-first template (do not start from a blank file).
  • Keep PG setup minimal: values, context, answer objects.
  • Write the prompt in PGML and attach answers to blanks.
  • Use a realistic science story and keep the story stable across randomizations.
  • Verify units, rounding expectations, and tolerance settings.
  • Preview multiple student versions to ensure randomization stays valid.

Finding and reusing questions

  • Use search to find existing questions close to what you want.
  • Favorite or clone questions you plan to reuse.
  • Clone before editing so you keep the original available as a reference.

Adding questions to assignments

  • Add questions from multiple sources (search, commons, your courses, your question area).
  • Reorder questions to match your intended learning progression.
  • Adjust points only after you have confirmed the question behavior.

Quality assurance (before students see it)

  • Check that every answer blank has a corresponding answer checker.
  • Check that random values never generate invalid biology or chemistry scenarios.
  • Check that the prompt renders cleanly without relying on external links.
  • Check accessibility: tables should be data tables, not layout hacks.

Non-WeBWorK question types

ADAPT courses often mix WeBWorK questions with other question formats. This chapter includes a question type cookbook with science examples (multiple choice, matching, ordering, and fill-in-the-blank) that can complement a PGML-first WeBWorK workflow.
