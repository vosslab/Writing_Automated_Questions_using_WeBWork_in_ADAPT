The same PGML techniques work across disciplines, but the best question banks feel like the subject: the prompt style, the data format, and the answer expectations match what students do in that field. This chapter collects "patterns" for science and descriptive courses, inspired by the types of prompts used in science-focused question banks.

This chapter assumes you already know the core question types from Chapter 5. The focus here is how to apply those question types to a specific subject area with realistic prompts, units, and data.

Life-science-first emphasis: many of the subsections below are designed to answer "what breaks when the content is biology?" and to highlight patterns that remain robust for grading.

General pattern: stable story, variable numbers

  • Keep the scenario fixed (one paragraph).
  • Randomize 2-4 values that drive the calculation.
  • Ask for one target quantity per blank.
  • Include a short units line so students know what to report.

Genetics

  • Counts and proportions: genotype counts in a table, ask for a probability or expected count.
  • Two-step prompts: compute a ratio, then interpret what it implies.
  • Multiple choice statements: one conceptual check after a numeric calculation.

Biochemistry

  • Enzyme kinetics: compute a velocity given parameters, with tolerance and unit handling.
  • pH and pKa: structured prompts with a small table of groups and pKa values.
  • Gel migration: rank-order or compute distance given a log-scale relationship.

Molecular biology

  • PCR setup: compute final concentrations from stock solutions.
  • Cloning logic: multiple choice about which construct is valid given constraints.
  • Expression analysis: interpret a small table of fold changes.

Biostatistics

  • Chi-squared: compute a test statistic from a contingency table.
  • Confidence intervals: compute a bound and select an interpretation statement.
  • Probability distributions: keep the parameters simple and the interpretation clear.

Design notes

  • Prefer tables and clearly labeled values over long prose.
  • Use PGML substitutions so variables render correctly inside math.
  • Use consistent rounding rules and make them explicit when needed.
  • Keep randomness biologically plausible (do not generate impossible scenarios).
