This page shows how to add visual emphasis using CSS-based styling that works in the PG 2.17 subset. Use this for terms like ABOVE/BELOW or NOT where color or bold weight prevents misreads.

Use this page when: you need to highlight key words or label classes in matching or statement-based prompts.

Preferred pattern (PGML tag wrapper)

BEGIN_PGML
[]{['span', style => 'color:#b91c1c; font-weight:700;']}{['','']}
END_PGML

TeX color commands do not render reliably in this install, so prefer HTML spans with simple CSS.

Passing HTML variables through PGML

If you build an HTML snippet in setup (for example, a colored label), pass it through PGML using [$var]* so it does not get escaped. 

$label = "ABOVE";

BEGIN_PGML
Threshold: [$label]*
END_PGML

Common failure and fix

Broken: [$label]
What you will see: raw <span> tags in the prompt
Fix: use [$label]* to prevent escaping

Apply it today

  • Use HTML spans for emphasis and keep the styling minimal.
  • Use [$var]* for HTML variables so PGML does not escape them.
