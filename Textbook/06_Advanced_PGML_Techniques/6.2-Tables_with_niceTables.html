This page explains how to build tables with niceTables.pl, which is the only supported way to create tables in this install. HTML table tags (table, tr, td, th) are blocked, so use DataTable or LayoutTable instead.

Use this page when: you need to present measurements, observations, or structured data in a readable format.

DataTable vs LayoutTable

Table type Use it for Notes
DataTable Real data with headers (measurements, conditions). Adds header structure and improves readability.
LayoutTable Visual layout or aligned blanks. Use sparingly; do not fake data tables.

Copy and edit (DataTable)

loadMacros(
  "PGstandard.pl",
  "MathObjects.pl",
  "PGML.pl",
  "niceTables.pl",
);

$rows = [
  ['Tube', 'Stock (mM)', 'Final (uM)'],
  ['A', 2.0, ''],
  ['B', 4.0, ''],
  ['C', 8.0, ''],
];

$tbl = DataTable($rows, columns => 'left', center => 1, horizontalrules => 1);

BEGIN_PGML
[@ $tbl @]*
END_PGML

Copy and edit (LayoutTable)

$layout = LayoutTable(
  [ 'Lane 1', 'Lane 2', 'Lane 3' ],
  [ '[________]{ $ans1 }', '[________]{ $ans2 }', '[________]{ $ans3 }' ],
  center => 1,
  horizontalrules => 1,
);

BEGIN_PGML
[@ $layout @]*
END_PGML

Biology examples to model

  • Dilution series: stock and final concentrations in a small DataTable.
  • Enzyme kinetics: substrate vs velocity table with a single computed blank.
  • Gel/blot comparison: LayoutTable with aligned labels and answer blanks.

Apply it today

  • Use DataTable for real measurements and LayoutTable only for layout.
  • Avoid raw HTML table tags; they are blocked in this install.
